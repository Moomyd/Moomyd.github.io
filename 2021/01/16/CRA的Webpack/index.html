发表于 2021-01-16 | 分类于 -Webpack

在实际的项目开发中，我们通常会使用脚手架来初始化项目。因为配置 Webpack 是一项繁复的工作，所以如果有一个工具可以自动生成配置，就可以大大加速项目的搭建。常用的脚手架，比如 Create React App。这款脚手架是用来初始化 React 项目的，但同样可以用来初始化普通的 JS 项目。

详解

create-react-app中是通过定义一个 Webpack 工厂函数来实现开发和产品环境区分的，通过之间传入对应的环境参数不同，生成不同环境的打包配置。

getStyleLoaders函数定义了处理css所需要的loaders

  • style-loader 通过注入<style>标签将CSS添加到DOM，建议将 style-loader 与 css-loader 结合使用。
  • css-loader 解释 @import 和 url() ，会 import/require() 后再解析它们，主要用于将 CSS 转换为JS模块。
  • postcss-loader 启用 postcss 来处理 Css，需要另外配置。通过配置不同插件，可以完成非常强大的功能。
  • sass-loader 加载 SASS / SCSS 文件并将其编译为 CSS。
  • resolve-url-loader 一个Webpack加载器，它基于原始源文件重写url（）语句中的相对路径。

  • dev:develop //开发模式，不进行优化提升开发效率
  • prd:production //生产模式，默认开启代码优化

  • strictExportPresence:使缺少的导出出现错误而不是警告
  • rules:
    • /.bmp/, /.gif/, /.jpe?g/, /.png/ url-loader解析

plugins

  • common
    • HtmlWebpackPlugin 打包到Html中
    • InterpolateHtmlPlugin 允许在html中加入变量

    • WorkboxWebpackPlugin pwa开启servicework
